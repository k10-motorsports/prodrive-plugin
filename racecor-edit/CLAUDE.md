# RaceCor Edit — AI Race Editing Pipeline

CLI tool that takes raw recordings from the RaceCor overlay (cockpit video + TV-view video + telemetry JSONL sidecar) and produces broadcast-quality race edits, condensed highlights, and social media clips.

## Tech Stack

- **Node.js ESM** — `"type": "module"`, requires Node ≥ 20
- **FFmpeg** — all video operations (segment extraction, concat, encode, silence detection)
- **Claude** (optional) — editorial refinement via `claude` CLI or `@anthropic-ai/sdk`
- **No build step** — run directly with `node bin/racecor-edit.js`

## Pipeline Architecture

```
ingest → analyze → render
          ├── scoring-engine (TV Score + Interest Score per second)
          ├── camera hysteresis (mechanical segments)
          ├── silence detection (boring signal)
          ├── claude-director (optional AI refinement)
          └── edl-generator → edit-decisions.json

condense = ingest + interest scoring + threshold adjustment + Claude + render
```

## Claude Integration (Dual-Mode)

The tool supports two Claude integration modes — users choose at runtime:

1. **CLI mode** (default) — pipes prompts to the `claude` CLI tool via stdin. Uses the user's Claude subscription. No API key needed.
2. **SDK mode** — uses `@anthropic-ai/sdk` with `ANTHROPIC_API_KEY`. Better for automation / CI.

Detection order:
- `RACECOR_CLAUDE_MODE=cli|sdk` → explicit override
- `ANTHROPIC_API_KEY` set → SDK mode
- `claude` on PATH → CLI mode
- Neither → mechanical-only (no AI refinement, still works)

The SDK (`@anthropic-ai/sdk`) is in `optionalDependencies` — if it's not installed, CLI mode still works. Both modes fall back gracefully to mechanical scoring if Claude is unavailable.

## Scoring System

Two parallel scoring engines run per-second over telemetry frames:

- **TV Score (0–100)**: Camera switching preference. High = show TV view (battles, overtakes). Low = stay on cockpit.
- **Interest Score (0–100)**: Viewer engagement. High = keep in condensed edit. Low = cut.

Both use weighted signal detection (gap proximity, position changes, incidents, flags, speed drops, clean air, etc.) with configurable thresholds and hysteresis to prevent rapid switching.

## Input Format

Expects a session directory containing:
- `.telemetry.jsonl` — sidecar from RaceCor overlay recording (one JSON object per frame)
- `.mp4` / `.webm` — cockpit and/or TV-view video files
- `session.json` — generated by `ingest` command

## Commands

```bash
racecor-edit ingest <dir>                    # Parse telemetry, detect events
racecor-edit analyze <dir>                   # Score + Claude → edit-decisions.json
racecor-edit preview <dir>                   # Low-res draft with timecodes
racecor-edit render <dir>                    # Full-res final edit
racecor-edit condense <dir> --target 5:00    # Smart-cut to target duration
racecor-edit clips <dir> --social            # Include 9:16 vertical exports
```
